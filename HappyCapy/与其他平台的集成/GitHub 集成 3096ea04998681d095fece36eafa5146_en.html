📝 Get GitHub Token

Token is like a key that allows HappyCapy to operate GitHub on your behalf. Getting it is simple:

Step 1: Open GitHub Token settings

Visit: https://github.com/settings/tokens

Or: On the GitHub page → avatar in the upper right corner → Settings → bottom left Developer Settings → Personal access tokens → Tokens (classic)

Step 2: Generate new Token

  1. Click "Generate new token" → select "Generate new token (classic)"
  1. Fill in Note (remarks), write whatever you want, such as "HappyCapy"
  1. It is recommended to select "No expiration" for Expiration.
  1. Select scopes:

Click the "Generate token" button at the bottom of the page to generate a string ofghp_the beginning of the string.

💡 Tips for use

1. Token only needs to be provided once

In the same conversation, after providing the Token for the first time, there is no need to provide it again for subsequent operations. HappyCapy will remember your Token.

2. You can abbreviate the warehouse name

If it is your own warehouse, you can just say the warehouse name instead of writing the full URL every time:

"帮我看看 my-project 的 Issue"

而不是：
"帮我看看 https://github.com/username/my-project 的 Issue"

❓ FAQ

Are Tokens safe?

The token is only used within your session and will not be stored or shared with others. suggestion:

  • Replace Token regularly (for example, every 3 months)
  • Do not share tokens in public
  • If leakage is suspected, immediately delete the old Token in GitHub and generate a new one

What should I do if I lose my Token?

It doesn't matter! If Token is lost, you can:

  1. Return to GitHub Token settings page
  1. Delete old token
  1. Regenerate a new one

It's that simple!
