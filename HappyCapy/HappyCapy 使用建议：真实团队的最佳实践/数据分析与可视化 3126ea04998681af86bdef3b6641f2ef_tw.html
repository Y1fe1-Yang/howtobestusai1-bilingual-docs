資料分析與可視化

適用人羣

資料科學家、資料分析師、商業智能專員、機器學習工程師

核心場景

📊 場景一：從零構建交互式儀表盤

真實案例： 資料科學團隊通過 AI 構建 5000 行 TypeScript 可視化應用，無需自己懂 JavaScript。

在 HappyCapy 中如何做：

"請幫我構建一個交互式儀表盤來分析使用者留存資料：

資料源：
- 使用者註冊資料（users.csv）
- 使用者活動日誌（activity.csv）

功能需求：
- 展示 7 天、30 天、90 天留存率曲線
- 按使用者來源（廣告、自然流量、推薦）分組對比
- 可以選擇時間範圍篩選
- 用柱狀圖展示每週活躍使用者趨勢

技術棧：用 React + Recharts，生成一個可以直接執行的網頁。"

HappyCapy 會：

  1. 讀取並處理你的資料檔案
  1. 寫完整的 React 應用程式碼
  1. 自動安裝依賴、設定開發環境
  1. 啓動本地服務器，生成預覽連結
  1. 你可以直接在瀏覽器中查看和交互

關鍵優勢：

  • ✅ 不需要懂前端開發
  • ✅ 程式碼可複用（下次分析類似資料直接修改）
  • ✅ 比 Jupyter Notebook 更持久、更易分享

時間節省：節省 2-4 倍時間

🔍 場景二：探索性資料分析（EDA）

在 HappyCapy 中如何做：

"請幫我分析這份銷售資料（sales_2024.csv）：
1. 給我一個資料概覽（行數、列數、缺失值）
2. 生成描述性統計（均值、中位數、標準差）
3. 識別異常值
4. 做相關性分析，看哪些因素影響銷售額
5. 畫出關鍵指標的分佈圖和趨勢圖
6. 總結 3-5 個關鍵發現"

HappyCapy 自動：

  • 📈 生成多種可視化圖表
  • 🧮 執行統計分析
  • 🔎 發現資料中的模式和異常
  • 📄 輸出結構化報告

🤖 場景三：機器學習模型訓練與評估

在 HappyCapy 中如何做：

"用這份客戶流失資料（churn_data.csv）訓練一個預測模型：

1. 資料預處理（處理缺失值、標準化數值特徵）
2. 特徵工程（生成有用的新特徵）
3. 訓練幾個模型（邏輯迴歸、隨機森林、XGBoost）
4. 對比模型性能（準確率、召回率、F1、AUC）
5. 生成特徵重要性分析
6. 給我最佳模型的預測程式碼，可以對新客戶做預測"

HappyCapy 協助你完成：

  • 🧪 自動特徵工程
  • 🎯 模型選擇與超參數調優
  • 📊 生成性能對比報告
  • 💾 保存訓練好的模型
  • 📝 輸出可直接使用的預測程式碼

📉 場景四：監控儀表盤資料異常

真實案例： 資料基礎設施團隊監控 200 個儀表盤，自動識別資料異常。

在 HappyCapy 中如何做：

"請幫我設置一個自動監控：
- 每天早上 9 點檢查這個 BigQuery 資料表
- 如果日活躍使用者比 7 天均值低 20%，就發警報
- 如果錯誤率超過 5%，就發警報
- 生成每日資料摘要，發送到我郵箱"

💡 給資料分析師的建議

1. 從一次性 Notebook 轉向持久工具

  • 不要每次都寫新的 Python 腳本
  • 讓 HappyCapy 構建可複用的 Web 儀表盤
  • 保存工作流，下次直接使用

2. 需要簡單時果斷打斷

  • AI 傾向於過度複雜的解決方案
  • 如果看到不必要的複雜程式碼，直接說："試試更簡單的方法"
  • HappyCapy 會立即調整

3. 跨語言無障礙

  • 你只需要懂資料分析概念，不需要精通各種編程語言
  • "用 Python 處理資料，用 JavaScript 做可視化" - HappyCapy 自動處理

4. 像用"老虎機"一樣使用

  • 保存當前工作
  • 讓 HappyCapy 自主工作 30 分鐘
  • 結果滿意就接受,不滿意就重新開始
  • 比手動修復 AI 的錯誤更高效
