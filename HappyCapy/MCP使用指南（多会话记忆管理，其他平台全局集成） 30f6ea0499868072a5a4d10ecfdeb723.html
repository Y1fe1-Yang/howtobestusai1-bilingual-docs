MCP使用指南（多会话记忆管理，其他平台全局集成）

HappyCapy 官网直通车：https://happycapy.ai/ 在这里，你可以看到机智可爱的 HappyCapy 如何成为你的 AI 伙伴，一起探索无限可能！

什么是 MCP？

MCP（Model Context Protocol，模型上下文协议）是一个开放的协议标准，让 AI 助手能够安全地连接到各种外部工具、数据源和服务。

简单来说，MCP 就像是给 AI 装上了“插件系统”，让它能够：

  • 📧 读写你的 Gmail 邮件
  • 💬 在 Slack 上发送消息
  • 📝 访问和编辑 Notion 页面
  • 🐙 管理 GitHub 仓库和 Issues
  • 🧠 在对话间持久化记忆知识图谱
  • 🔗 连接 1000+ 其他应用和服务

为什么需要 MCP？

在 HappyCapy 中安装 MCP 后，你可以：

  • 自动化日常工作 - 让 AI 帮你处理邮件、安排日程、整理文档
  • 跨平台协作 - 在一个对话中完成多个应用的操作（比如从 Gmail 读取附件，处理后上传到 Notion）
  • 持久化上下文 - 通过 Memory MCP，AI 能记住项目信息、人物关系等重要内容，跨会话保持记忆
  • 简化开发流程 - 直接在对话中管理 GitHub 仓库、创建 Issue、查看代码

🚀 快速安装：使用我们的自动化技能

我们为 HappyCapy 开发了一个一键安装 MCP 的技能，可以自动帮你配置 Memory MCP、Composio MCP 和 GitHub Copilot MCP。

第 1 步：安装技能

在 HappyCapy 对话中输入：

安装https://github.com/Y1fe1-Yang/happycapy-mcp-manager技能

第 2 步：运行技能

安装完成后输入：

/happycapy-mcp-manager

或者直接说：

🧠 重点推荐：Memory MCP

Memory MCP 是一个官方的知识图谱存储 MCP，能为 AI 提供持久化的记忆能力。这是我们最推荐的 MCP！

为什么需要 Memory MCP？

  • 跨会话记忆 - AI 能记住你的项目信息、代码架构、人物关系等，下次对话时不用重复介绍
  • 知识图谱 - 以实体和关系的形式存储信息，更结构化、更好搜索
  • 长期上下文 - 对于长期项目，AI 能持续积累对项目的理解

💡 Memory MCP 使用示例

安装 Memory MCP 后，您可以使用以下 Prompt 来集成历史对话信息到知识图谱中。根据您的需求选择合适的版本：

1️⃣ 完整版（适合首次使用）

详细扫描所有历史 session，提取项目、技术栈、能力画像等信息，适合建立完整的知识图谱。

把整个系统我的所有历史对话信息集成到 Memory MCP 知识图谱中。

具体要求：
1. 扫描本主机下所有历史 session 的工作空间
2. 分析每个工作空间的项目内容、技术栈、创建时间
3. 提取关键信息并提炼成结构化的背景知识
4. 将以下信息存储到知识图谱：
   - 我完成的所有项目
   - 使用过的技术和工具
   - 项目与技术之间的关系
   - 我的技术能力画像和偏好
   - 时间线和活跃周期
5. 最后生成一份完整的总结报告

2️⃣ 简化版（推荐）

快速提取历史对话的核心信息，适合大多数用户的日常使用。

把本主机下所有历史 session 的对话信息提炼成背景知识，集成到 Memory MCP 知识图谱中。

3️⃣ 快速版（最简洁）

最简单直接的表达，适合快速使用。

把之前所有 session 的对话提炼成记忆存到知识图谱

💡 提示：复制上述任意一个代码块中的 Prompt，直接粘贴到对话框即可开始使用。Happycapy 会自动扫描历史 session 并将信息整合到 Memory MCP 知识图谱中。

💡 提示：安装Memory之后的对话将自动保存，无需重复此命令，该服务于沙箱内自行完成，Happycapy不会使用任何您的信息
