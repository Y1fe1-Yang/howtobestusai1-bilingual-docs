使用本地 IDE(以VS Code為例) 連接 HappyCapy沙箱

HappyCapy 官網：https://happycapy.ai/

💡 適用場景

有些開發場景下，你可能需要使用本地 IDE：

  • 大型專案開發（數百個檔案）
  • 使用已配置好的本地開發環境
  • 需要特定的付費擴充或企業工具
  • 習慣本地 IDE 的快捷鍵與工作流

好消息：透過 VSCode Remote Tunnels，可輕鬆連接到 HappyCapy 雲端沙箱！

✅ 準備工作

  • HappyCapy 帳號（訪問 https://happycapy.ai/）
  • 本地 VSCode 1.70+
  • GitHub 帳號（用於授權）
  • 網路連線

🚀 開始連接（3 步完成）

步驟 1️⃣：在 HappyCapy 啟動 Tunnel

在 HappyCapy 對話框中輸入：

請幫我啟動 VSCode Remote Tunnel（全局模式）--name happycapy，並展示 GitHub 認證連結與驗證碼

HappyCapy會自動安裝並啟動 Tunnel，然後回傳：

  • 授權頁面：https://github.com/login/device
  • 授權碼：XXXX-XXXX（8 位代碼）

步驟 2️⃣：GitHub 授權（1 分鐘）

  1. 1. 開啟瀏覽器，訪問：https://github.com/login/device
  1. 2. 輸入授權碼
  1. 3. 點擊 [Authorize] 授權

步驟 3️⃣：本地 VSCode 連接

安裝擴充

  1. 1. 打開本地 VSCode
  1. 2. 按 Ctrl + Shift + X 開啟擴充面板
  1. 3. 搜索 Remote - Tunnels（Microsoft 官方）
  1. 4. 點擊 Install

連接 Tunnel

  1. 按 F1 開啟命令面板
  1. 輸入：Remote-Tunnels: Connect to Tunnel

3. 登入 GitHub（首次需要）

4. 選擇 happycapy

等待連接...

📂 開始使用

打開遠端資料夾

  1. 1. 點擊 "Open Folder..."
  1. 2. 輸入路徑：/home/node 或 /workspace
  1. 3. 點擊 "Yes, I trust the authors"

開啟終端機

按 Ctrl + ` 開啟遠端終端機

❓ 常見問題

Q: 找不到我的 Tunnel？

  • 1. 在 HappyCapy 中問：「VSCode Tunnel 還在運行嗎？」
  • 2. 如果沒有運行，重新輸入啟動 prompt
  • 3. 檢查本地網路連線
  • 4. 重新登入 GitHub：F1 → Remote-Tunnels: Sign out

Q: 連接逾時

  • 1. 檢查網路連線
  • 2. 暫時關閉 VPN
  • 3. 在 HappyCapy 中重啟 Tunnel
  • 4. 更新 VSCode 到最新版

Q: 檔案無法儲存

  • 1. 檢查磁碟空間：在終端機執行 df -h
  • 2. 檢查檔案權限：ls -la
  • 3. 重新連接 Tunnel
  • 日常可用 HappyCapy 瀏覽器端快速開發；遇到複雜專案再切換到本地 VSCode。兩者共享同一個雲端沙箱，檔案無需遷移！
