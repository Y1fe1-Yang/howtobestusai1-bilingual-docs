Product design and prototyping

Applicable people

Product Designer, UI/UX Designer, Product Manager

core scene

🎨 Scenario 1: From Mockup to functional prototype

Real case:The design team directly pastes the mockup screenshots to generate interactive front-end code.

How to do it in HappyCapy:

  1. Design the interface in Figma/Sketch
  1. Take a screenshot and paste it into HappyCapy（Cmd+V / Ctrl+V）
  1. Describe interaction logic

HappyCapy will:

  • Identify elements in Mockup (buttons, input boxes, layouts)
  • Generate complete React component code
  • Implement interaction logic
  • Start the development server and provide a preview link

Key advantages:

  • ✅ Designers don’t need to write code
  • ✅ Engineers directly see the working prototype
  • ✅ Reduce communication costs between design and development
  • ✅ Quickly verify interaction logic

Time savings: 2-3 times faster execution

🔄 Scenario 2: Quick visual adjustment (color, font, spacing)

Real case:The design team no longer writes design documents and implements visual changes directly through AI Code.

How to do it in HappyCapy:

"Help me adjust the visual style of the login page:

File: src/pages/Login.tsx

Change requirements:
- Change primary color from blue (#3B82F6) to purple (#8B5CF6)
- Change the title font from 'Inter' to 'Poppins', weight 600
- Change input height from 40px to 48px
- Change button border radius from 4px to 8px
- Increase overall spacing by 20% for a more breathable layout

Keep existing functionality unchanged; style updates only."

HappyCapy will:

  • Precise modification of CSS/Tailwind classes
  • Keep all functional logic
  • Automatically run the front end and generate a preview
  • You can see the effect immediately and submit it when you are satisfied

Workflow changes:

  • ❌ Old process: Write design documents → Engineers understand → Implement → Feedback → Modify
  • ✅ New process: Designer implements directly → Preview → Complete

💡 Advice for product designers

1. Ask engineers to help with initial setup

  • Git permission configuration
  • Development environment settings
  • Once configured, it’s a revolutionary workflow

2. Boot HappyCapy with custom memory file

  • In the conversation say: "Remember this: I'm a designer with limited programming experience who needs detailed explanations and smaller incremental changes"
  • HappyCapy will adjust communication methods to be more friendly

3. Make extensive use of picture paste

  • Cmd+V / Ctrl+V paste the screenshot directly
  • HappyCapy is good at "reading pictures" to generate code
  • 10 times faster than describing an interface with text

4. Start with simple tasks

  • Try first: change button color
  • Then try: Adjust the entire page layout
  • Last try: Prototyping complex interactions
  • Build confidence step by step
