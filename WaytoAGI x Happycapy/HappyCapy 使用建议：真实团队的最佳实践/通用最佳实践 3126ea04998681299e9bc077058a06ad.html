通用最佳实践

✅ 高效使用 HappyCapy 的黄金法则

1. 清晰描述你的目标，而非过程

❌ 不好的提示：

"先用 pandas 读取 CSV，然后用 matplotlib 画图"

✅ 好的提示：

"分析这份销售数据，生成趋势图，告诉我关键发现"

为什么？ HappyCapy 会自动选择最佳工具和方法，你只需要说明目标。

2. 提供足够的上下文

❌ 不好的提示：

"帮我修复这个 bug"

✅ 好的提示：

"用户点击登录按钮后没有反应。

环境：React 18 + TypeScript
文件：src/components/LoginForm.tsx
浏览器控制台显示：'Cannot read property of undefined'
预期行为：点击后应该发送 API 请求并跳转到首页

请帮我定位问题并修复。"

3. 迭代式工作，不要要求一次性完美

❌ 不好的提示：

"帮我构建一个完整的电商网站，包含用户认证、产品列表、购物车、支付集成、订单管理、评论系统、推荐算法..."

✅ 好的提示：

"先帮我搭建基本框架：
- 首页展示产品列表
- 产品详情页
- 简单的购物车（无需后端）

完成后我们再逐步添加功能。"

原则： 拆解大任务 → 逐步实现 → 每步验证

4. 利用 HappyCapy 的多模态能力

你可以：

  • 📷 粘贴截图（Cmd/Ctrl + V）
  • 📎 上传文件（PDF、Excel、Word、图片）
  • 🔗 分享网页链接
  • 🎤 提供录音或视频（转写后分析）

5. 保存成功的工作流

当一个提示效果很好时：

  • 📝 记录下来（复制到笔记）
  • 🏷️ 标注使用场景
  • 🔄 下次直接复用或微调

Pro Tip： 你可以把成功的工作流自定义为技能（Skill），所有用户都可以这样做！我们鼓励你创建自己的技能库，让重复任务一键执行。Pro/Max 用户还可以设置定时自动化。

6. 定期检查点，方便回滚

对于长时间任务或实验性工作：

"在实现这个功能前，先让我确认当前代码都保存好了"
  • 如果使用 Git：定期提交
  • 如果在 HappyCapy 工作区：定期下载备份

像玩游戏一样存档，出错时可以轻松回滚。

7. 明确你的技术水平

在对话开始时说明：

"我是设计师，不太懂编程，请用简单语言解释"

或

"我是高级开发者,熟悉 React 和 TypeScript，可以直接给我代码"

HappyCapy 会调整沟通方式和细节程度。

8. 不要害怕"重新开始"

如果对话方向跑偏了：

  • 不要试图纠正 AI 的错误方向
  • 直接开新对话，重新描述任务
  • 通常比修正更快

像使用"老虎机"：

  • 保存状态
  • 尝试 30 分钟
  • 满意就接受，不满意就重来

9. 利用 HappyCapy 的技能系统

HappyCapy 支持使用开源社区的所有技能，涵盖：

  • 🎨 图像/视频生成
  • 📄 文档处理（PDF、Word、Excel、PPT）
  • 🌐 网页开发
  • 📊 数据分析
  • 🔬 学术研究
  • 📧 邮件自动化
  • 🎥 视频剪辑
  • ...更多

使用方法：

"用 ai-image-generation 技能帮我生成一张科技感海报"
"用 comprehensive-researcher 技能研究'大语言模型的最新进展'"

查看所有可用技能：在 HappyCapy 对话中输入 /help

10. Max Plan 用户：拥抱"无限"心态

如果你使用 Max Plan：

  • ✅ 不要担心 token 用量
  • ✅ 大胆尝试复杂任务
  • ✅ 让 HappyCapy 自主工作更长时间
  • ✅ 多次迭代直到满意

这就是 Max Plan 的价值：零焦虑使用 AI。

🚫 常见误区与避免方法

误区 1："AI 应该一次性给我完美结果"

现实： AI 是协作伙伴，不是魔法棒。

正确心态：

  • 迭代式工作
  • 提供反馈
  • 逐步改进

误区 2："我的提示不够'专业'，AI 不会理解"

现实： 自然语言就是最好的提示。

正确方法：

  • 像和朋友聊天一样描述需求
  • 不需要学"提示工程"
  • HappyCapy 会理解你的意图

误区 3："我不懂技术，不能用 AI 编程工具"

现实： HappyCapy 专为非技术人员设计。

证据：

  • 财务团队零编程经验，自己执行复杂数据工作流
  • 法律团队无开发背景,一小时构建无障碍应用
  • 产品设计师直接实现前端功能

🎯 不同角色的快速上手路径

非技术人员（3 步上手）

  1. 第一月： 构建自验证循环，让 HappyCapy 长时间自主工作

  1. 第一月： 把常用分析工作流保存为可复用工具
