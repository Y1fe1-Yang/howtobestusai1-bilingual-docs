General best practices

✅ Golden Rules for Effective Use of HappyCapy

1. Clearly describe your goals, not the process

❌ Bad tips:

"First use pandas to read the CSV, then use matplotlib to draw the graph"

✅ Good tips:

"Analyze this sales data, generate a trend chart, and tell me the key findings"

Why? HappyCapy automatically selects the best tools and methods, you just need to state the goal.

2. Provide sufficient context

❌ Bad tips:

"Help me fix this bug"

✅ Good tips:

"There is no response after the user clicks the login button.

Environment: React 18 + TypeScript
File: src/components/LoginForm.tsx
The browser console shows: 'Cannot read property of undefined'
Expected behavior: After clicking, an API request should be sent and jump to the homepage

Please help me locate the problem and fix it. "

3. Work iteratively, don’t ask for one-time perfection

❌ Bad tips:

"Help me build a complete e-commerce website, including user authentication, product list, shopping cart, payment integration, order management, review system, recommendation algorithm..."

✅ Good tips:

"Help me build the basic framework first:
- Home page displays product list
- Product details page
- Simple shopping cart (no backend required)

After completion, we will gradually add functions. "

Principle: Dismantle big tasks → implement step by step → verify each step

4. Leverage the multi-modal capabilities of HappyCapy

you can:

  • 📷 Paste screenshot (Cmd/Ctrl + V)
  • 📎 Upload files (PDF, Excel, Word, pictures)
  • 🔗 Share web link
  • 🎤 Provide audio or video (analyzed after transcription)

5. Save successful workflow

When a tip works well:

  • 📝 Record it (copy to notes)
  • 🏷️ Mark usage scenarios
  • 🔄 Reuse or fine-tune directly next time

Pro Tip: You can customize successful workflows as skills, and all users can do this! We encourage you to create your own skill library to perform repetitive tasks with one click. Pro/Max users can also set up scheduled automation.

6. Regular checkpoints to facilitate rollback

For long tasks or experimental work:

"Before implementing this function, let me confirm that the current code has been saved."
  • If using Git: commit regularly
  • If in HappyCapy workspace: Download backups regularly

Save like a game and easily roll back when something goes wrong.

7. Identify your skill level

At the beginning of the conversation state:

"I'm a designer and don't know much about programming. Please explain in simple language."

or

"I am a senior developer, familiar with React and TypeScript, you can give me code directly"

HappyCapy will adjust communication style and level of detail.

8. Don’t be afraid to “start over”

If the conversation goes astray:

  • Don’t try to correct the AI’s mistakes
  • Directly open a new conversation and re-describe the task
  • Usually faster than correction

Like using "slot machine":

  • save state
  • Try it for 30 minutes
  • If you are satisfied, accept it; if you are not satisfied, try again.

9. Utilize HappyCapy’s skill system

HappyCapy supports the use of all skills in the open source community, covering:

  • 🎨 Image/video generation
  • 📄 Document processing (PDF, Word, Excel, PPT)
  • 🌐 Web Development
  • 📊 Data analysis
  • 🔬 Academic research
  • 📧 Email automation
  • 🎥 Video clips
  • ...More

How to use:

"Use ai-image-generation skills to help me generate a technological poster"
"Use comprehensive-researcher skills to study 'Latest Progress in Large Language Models'"

View all available skills: Enter in HappyCapy dialogue/help
