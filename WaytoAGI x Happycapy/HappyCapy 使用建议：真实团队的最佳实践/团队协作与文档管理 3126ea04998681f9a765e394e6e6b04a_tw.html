團隊協作與檔案管理

核心場景

📖 場景一:自動生成和更新檔案

真實案例: 安全團隊通過 AI 綜合多個檔案,建立 Runbook 和故障排查指南。

在 HappyCapy 中如何做:

"請幫我建立一份'新員工入職指南':

輸入檔案:
- 公司政策檔案(policy.pdf)
- 技術棧介紹(tech_stack.md)
- 常見問題 FAQ(faq.docx)
- 團隊聯繫方式(contacts.xlsx)

輸出要求:
- Markdown 格式,清晰的章節結構
- 提取關鍵資訊,去除冗餘
- 添加目錄和快速索引
- 每個章節末尾添加"延伸閱讀"連結
- 最後生成 PDF 版本"

HappyCapy 會:

  • 讀取所有輸入檔案
  • 提取和組織資訊
  • 生成結構化檔案
  • 輸出 Markdown 和 PDF 版本

📊 場景二:會議紀要和行動項管理

在 HappyCapy 中如何做:

"這是我們今天產品會議的錄音轉寫:
[粘貼轉寫文本]

請:
1. 總結會議的關鍵討論點
2. 提取所有行動項(Action Items),標註負責人和截止日期
3. 識別未解決的問題
4. 生成結構化的會議紀要
5. 建立一個任務追蹤表格(CSV 格式),可導入 Asana/Jira"

HappyCapy 會:

  • 分析會議內容
  • 提取關鍵資訊
  • 生成易讀的紀要
  • 建立可操作的任務清單

💡 團隊協作建議

1. 分享成功的工作流

  • 團隊成員互相演示如何使用 HappyCapy
  • 發現新的使用方法
  • 建立團隊最佳實踐庫

2. 建立團隊知識庫

  • 把常用提示詞保存在團隊檔案或者優化爲Skill
  • 新人可以直接安裝使用
  • 持續優化和迭代

3. 會話結束時更新檔案

  • 每次使用 HappyCapy 完成任務後
  • 讓它總結工作流程
  • 更新團隊檔案,供下次使用
