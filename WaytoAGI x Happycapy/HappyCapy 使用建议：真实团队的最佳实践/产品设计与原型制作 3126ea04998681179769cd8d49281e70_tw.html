產品設計與原型製作

適用人羣

產品設計師、UI/UX 設計師、產品經理

核心場景

🎨 場景一：從 Mockup 到功能原型

真實案例： 設計團隊直接粘貼 Mockup 截圖，生成可交互的前端程式碼。

在 HappyCapy 中如何做：

  1. 在 Figma/Sketch 中設計介面
  1. 截圖並粘貼到 HappyCapy（Cmd+V / Ctrl+V）
  1. 描述交互邏輯

HappyCapy 會：

  • 識別 Mockup 中的元素（按鈕、輸入框、佈局）
  • 生成完整的 React 組件程式碼
  • 實現交互邏輯
  • 啓動開發服務器，提供預覽連結

關鍵優勢：

  • ✅ 設計師不需要寫程式碼
  • ✅ 工程師直接看到可執行的原型
  • ✅ 減少設計-開發之間的溝通成本
  • ✅ 快速驗證交互邏輯

時間節省：執行速度提升 2-3 倍

🔄 場景二：快速視覺調整（顏色、字體、間距）

真實案例： 設計團隊不再寫設計檔案，直接通過 AI Code 實現視覺變更。

在 HappyCapy 中如何做：

"請幫我調整登入頁面的視覺樣式：

檔案：src/pages/Login.tsx

變更需求：
- 主色調從藍色（#3B82F6）改為紫色（#8B5CF6）
- 標題字體從 'Inter' 改為 'Poppins'，加粗 600
- 輸入框高度從 40px 改為 48px
- 按鈕圓角從 4px 改為 8px
- 整體間距增加 20%，讓頁面更透氣

保持現有功能不變，只改樣式。"

HappyCapy 會：

  • 精確修改 CSS/Tailwind 類
  • 保留所有功能邏輯
  • 自動執行前端，生成預覽
  • 你可以立即看到效果，滿意後提交

工作流變化：

  • ❌ 舊流程：寫設計檔案 → 工程師理解 → 實現 → 反饋 → 修改
  • ✅ 新流程：設計師直接實現 → 預覽 → 完成

💡 給產品設計師的建議

1. 請工程師幫助初始設置

  • Git 權限設定
  • 開發環境設置
  • 一旦設定完成，就是革命性的工作流

2. 用自定義記憶檔案引導 HappyCapy

  • 在對話中說：“記住這個：我是設計師，編程經驗有限，需要詳細解釋和更小的增量變更”
  • HappyCapy 會調整溝通方式，更友好

3. 大量使用圖片粘貼

  • Cmd+V / Ctrl+V 直接粘貼截圖
  • HappyCapy 擅長“讀圖”生成程式碼
  • 比用文字描述介面快 10 倍

4. 從簡單任務開始

  • 先嚐試：修改按鈕顏色
  • 然後嘗試：調整整個頁面佈局
  • 最後嘗試：構建複雜交互原型
  • 逐步建立信心
